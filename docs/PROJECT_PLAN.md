# PROJECT_PLAN.md — CareerFit AI 프로젝트 기획

## 1. 문제 정의

취업 또는 공모전을 준비하면서 가장 불편했던 점은 무엇인가?

- 내가 가지고 있는 스택을 활용할 수 있는 **공모전**이 언제 어디에 나와 있는지 일일이 찾기가 불편하다.
- 내 전공·스킬과 맞는 **채용 공고**를 여러 사이트에서 따로 검색해야 해서 시간이 많이 든다.
- AI 추천 결과의 **근거와 출처**를 확인하기 어렵다.

> **범위 조정:** 초기 기획에 포함됐던 대학원·연구실 추천 기능은 데이터 수집 난이도로 인해 **본 프로젝트 범위에서 제외**한다.  
> 현재는 **채용 공고(jobs)** 와 **공모전(competitions)** 데이터만 다룬다.

---

## 2. 사용자 정의

### 대상

- 취업을 준비 중인 대학교 3~4학년 학생
- 보유 스택은 있지만, 맞는 공모전·채용 공고를 찾기 어려운 사용자
- 공모전·채용 마감을 놓치고 싶지 않은 사용자

### 사용 시나리오

#### 시나리오 1) 맞춤형 공모전 추천

1. 사용자는 전공, 학년, 보유 기술(Python, AI, 디자인 등), 관심 분야(AI, 데이터 분석 등)를 입력한다.
2. CareerFit AI는 사용자의 역량과 관심 분야를 분석한 후 **공모전 데이터베이스**(`competitions.csv`)를 검색한다.
3. AI는 사용자의 역량과 가장 높은 적합도를 가진 공모전을 추천하고, 각 공모전의 모집 기간, 참가 자격, 주요 키워드, 예상 적합도(%)를 함께 제공한다.
4. 사용자는 관심 있는 공모전을 선택하여 상세 정보와 공식 공고 링크를 확인한 후 바로 지원할 수 있다.

#### 시나리오 2) 맞춤형 채용 공고 추천

1. 사용자는 전공, 보유 스킬, 관심 직무(예: 데이터 분석, AI/ML)를 입력한다.
2. CareerFit AI는 **채용 공고 데이터베이스**(`jobs.csv`)에서 관련 공고를 검색한다.
3. AI는 필수·우대 스킬, 직무 설명, 마감일을 바탕으로 적합한 공고를 추천한다.
4. 사용자는 추천된 공고의 상세 정보를 확인하고 지원을 준비할 수 있다.

#### 시나리오 3) AI 맞춤 분석 및 추천 근거 제공

1. 사용자가 `/analyze` API에 프로필 정보를 전송한다.
2. CareerFit AI는 RAG로 검색된 jobs·competitions 데이터와 Gemini API를 활용해 맞춤 조언을 생성한다.
3. 추천에 활용된 데이터 출처(`sources`)를 함께 제시한다.
   - 채용 공고 데이터 (`jobs.csv`)
   - 공모전 공식 정보 (`competitions.csv`)
   - 정부·공공기관 공모전 데이터
4. 사용자는 AI 추천의 신뢰성을 확인하고 필요한 정보를 직접 검증할 수 있다.

## 3. 데이터 계획

### (1) 사용자 프로필 데이터

사용자가 입력하는 정보로 AI 추천의 기준이 되는 데이터이다.

| 컬럼명 | 설명 |
|---|---|
| 전공 | 사용자의 전공 |
| 학년 | 3학년, 4학년 등 |
| 기술 스택 | Python, SQL, AI 등 |
| 관심 분야 | AI, 데이터 분석 등 |
| 관심 직무 | 데이터 분석, AI/ML 등 |

### (2) 채용 공고 데이터 — `backend/data/jobs.csv`

사용자 스킬과 직무 적합도를 계산하는 데 활용한다.

| 컬럼명 | 설명 |
|---|---|
| id | 공고 고유 ID |
| company | 회사명 |
| title | 직무명 |
| required_skills | 필수 스킬 |
| preferred_skills | 우대 스킬 |
| description | 업무 설명 |
| job_type | 직무 유형 (데이터 분석, AI/ML 등) |
| deadline | 마감일 |

### (3) 공모전 데이터 — `backend/data/competitions.csv`

공모전 정보를 저장하여 사용자와의 적합도를 계산하는 데 활용한다.

| 컬럼명 | 설명 |
|---|---|
| id | 공모전 고유 ID |
| name | 공모전 이름 |
| organizer | 주최기관 |
| field | 분야 (AI, 데이터 분석 등) |
| required_skills | 요구 기술 |
| eligibility | 참가 자격 |
| prize | 상금 정보 |
| deadline | 마감일 |

### (4) 데이터 출처

| 데이터 | 출처 |
|---|---|
| 채용 공고 | 프로젝트용 CSV (`jobs.csv`, 18건) |
| 공모전 | 정부·공공기관 및 공모전 공식 홈페이지 기반 CSV (`competitions.csv`) |
| 사용자 데이터 | 사용자가 직접 입력한 정보 |

## 4. 기능 목록

| 기능 | 설명 | 상태 |
|---|---|---|
| `GET /health` | 서버 상태 확인 | ✅ 완료 |
| `GET /jobs` | 채용 공고 목록 조회 | ✅ 목업 (CSV 연동 예정) |

| 데이터 전처리 | jobs.csv Pandas 정제 | 🔄 진행 중 |
| SQLite 저장 | 정제된 데이터 DB 저장 | ⬜ 예정 |
| ChromaDB RAG | jobs·competitions 벡터 검색 | ⬜ 예정 |
