# EVAL_QUESTIONS.md — CareerFit AI 자기 평가 질문



## 이 질문들에 스스로 답할 수 있어야 최종 발표와 면접에 나설 수 있다.



## 프로젝트 전체

- 이 서비스가 해결하는 문제는 무엇인가?
취업 공고와 공모전을 쉽게 나열해서 확인할 수 있도록 해주고자 한다.
- RAG를 사용하지 않으면 어떤 문제가 생기는가?
LLM모델은 Hallucination을 일으켜 사용자에게 잘못된 정보를 제공하는데 RAG를 통해서 이 리스크 부담을 줄여줄 수 있다.
- 가장 어려웠던 구현은 무엇이고, 어떻게 해결했는가?
도커를 활용해 서버를 띄우는 과정이 처음이어서 서툴었던 것 같다. 하지만 차근차근 웹 구조를 생각하면서 하니 해결이 되었다.


## 데이터 관련

- 왜 이 데이터를 선택했는가?
취업과 공모전의 데이터를 활용하여 사용자에게 맞춤 정보를 제공하기 위함이다.
- 데이터의 어떤 부분이 신뢰할 수 없을 수 있는가?
아무래도 크롤링을 해서 데이터를 가져오는 경우 범위 설정에 따라서 관련없는 형식들의 데이터가 들어올 수도 있고, 결측치가 생기는 경우 이를 적절하게 처리하지 않으면 신뢰할 수 없다고 생각한다.
- SQLite와 ChromaDB에 같은 데이터를 저장하는 이유는 무엇인가?
SQLite에서 조건을 검색하는 데이터베이스 용도로 사용하고 ChromaDB에서 임베딩 유사도 검색을 하기 위함이다.
- 결측치와 중복을 어떻게 처리했는가?
중복의 경우 한 행을 제거 했고, 결측치의 경우도 그 행을 찾아 지우도록 하였다.


## RAG 관련

- RAG 구조에서 ChromaDB는 어떤 역할을 하는가?
RAG구조에서 ChromaDB는 데이터의 유사도 검색을 하는 역할을 한다.
- n_results 값을 높이면 어떤 트레이드오프가 생기는가?
높이면 노이즈가 증가하고, 토큰과 비용이 증가할 수 있고, 그로인해 답변 품질의 저하가 생긴다.
- 검색 결과 품질을 어떻게 검증했는가?

- Hallucination이 무엇이며, RAG가 이를 어떻게 줄이는가?
Hallucination은 LLM이 사용자에게 사실이 아닌 내용을 그럴듯하게 생성해주는 현상이다. RAG는 답변을 생성하기 전에 외부 지식(DB)에서 관련 정보를 먼저 검색하고 그 검색 결과를 답변으로 생성한다.
## API 관련

- /analyze 엔드포인트가 POST를 사용하는 이유는 무엇인가?
사용자가 입력한 데이터를 request body에 담아서 서버로 전달해야하기 때문이다. 
- 응답 JSON에 sources를 포함시킨 이유는 무엇인가?
RAG는 검색된 문서를 근거로 답변을 생성하기 때문이다.
- /health 엔드포인트가 왜 필요한가?
서버가 정상적으로 실행이 되는지 확인을 할 수 있도록 해주기 때문이다.


## 보안 관련

- .env 파일을 GitHub에 올리면 어떤 일이 생기는가?
Key들이 노출이 되어 외부 사용자들이 이를 보고 악용할 수 있다.
- .env와 .env.example의 차이는 무엇인가?
.env는 실제 key 값들이 들어가고 .env.example은 .env에 어떤식으로 들어가야하는지 예시를 작성해주는 역할을 한다.


## Docker 관련

- Docker를 사용하는 이유는 무엇인가?
애플리케이션과 실행환경을 하나의 컨테이너로 감싸 어디에서나 동일한 환경을 유지하여 실행할 수 있도록 하기 위함이다. 개개인의 버전이나 운여체제가 달라서 생기는 오류들을 줄이기 위함이다.
- Dockerfile에서 EXPOSE와 CMD의 역할은 무엇인가?
expose는 컨테이너가 사용할 포트 번호를 명시하는 역할이다.
컨테이너가 시작될 때 기본적으로 실행할 명령어를 지정한다.
